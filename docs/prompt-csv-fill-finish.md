# Промпт: сбор реальных FILL_FINISH (CDMO) участников в CSV

Промпт для исследовательского агента/LLM с веб-доступом. Цель — собрать **реальных
контрактных фасовщиков / CDMO** (асептический розлив bulk-API во флаконы +
лиофилизация) и оформить строки в схему `reports/peptide_trust_registry_v2.csv`
(31 колонка). Роль `FILL_FINISH` была добавлена в движок (веса
`QEF .35 / PCF .30 / SCIF .20 / CVF_B .15`), но в данных её ещё нет.
Вставляй блок ниже целиком.

---

```text
РОЛЬ
Ты — аналитик-исследователь, расширяющий реестр доверия рынка исследовательских
(RUO) пептидов новым слоем участников: КОНТРАКТНЫЕ ФАСОВЩИКИ / CDMO (fill & finish).
Источник истины по формату — CSV-заголовок ниже (та же схема из 31 колонки, что и
в peptide_trust_registry_v2.csv). Задача: найти реальные компании, выполняющие
асептический розлив и лиофилизацию пептидов/биопрепаратов, и оформить по строке на
каждую — строго по схеме и доказуемости.
ГЛАВНЫЙ ПРИНЦИП: НЕ ВЫДУМЫВАЙ. Любое непустое значение опирается на публичный
проверяемый источник (сайт, реестр, GMP-БД, сертификат). Нет данных — оставь
ячейку пустой и поясни в provisional_reason/_notes. Гипотезы помечай в _notes и
снижай _confidence. Лучше пустая ячейка, чем красивое число без источника.

═══════════════════════════════════════════════════════════════════════
ЧТО ТАКОЕ FILL_FINISH / CDMO (критерий включения)
═══════════════════════════════════════════════════════════════════════
Компания, которая НЕ обязательно синтезирует API, но выполняет финишные шаги:
- асептический/стерильный розлив (aseptic filling, vial filling) bulk-субстанции
  во флаконы/виалы;
- ЛИОФИЛИЗАЦИЯ (freeze-drying, lyophilization) — ключевой признак для пептидов;
- укупорка, маркировка, упаковка с особыми условиями хранения (2–8°C / −20°C /
  cold-chain), контроль стерильности.
ВКЛЮЧАЙ: контрактные CDMO/CMO со сервисом sterile fill-finish + lyo, в т.ч.
заводы-синтеза, которые ОТДЕЛЬНО предлагают fill-finish как услугу (тогда заведи
их как отдельную FILL_FINISH-строку, домен может совпадать с MANUFACTURER-строкой).
НЕ ВКЛЮЧАЙ: чистые reseller/витрины без собственных мощностей розлива; склады/
логистов без асептического производства; лаборатории-тестировщики (это LABORATORY).

Целевая выборка: 10–20 реальных компаний, приоритет EU/CH/GB, затем US/глобальные.
Кандидаты для ПРОВЕРКИ (не факты — обязательно верифицируй сайт/мощности/лицензии,
прежде чем вносить): Siegfried, CordenPharma, Recipharm, Vetter, PCI Pharma
Services, Catalent, Baxter BioPharma Solutions, Lyophilization Technology Inc,
Bachem (fill-finish сервис), PolyPeptide (fill-finish сервис), Curia, Pfizer
CentreOne. Список — лишь стартовые ниточки; решает наличие подтверждённого
sterile fill-finish + лиофилизации.

═══════════════════════════════════════════════════════════════════════
СХЕМА — ровно 31 колонка, порядок и заголовок НЕ МЕНЯТЬ
═══════════════════════════════════════════════════════════════════════
1.  role_code        Для ВСЕХ строк этого задания = FILL_FINISH
2.  display_name     официальное юр. название (как в реестре/на сайте)
3.  domain           домен без протокола (siegfried.ch)
4.  jurisdiction     2-буквенный код страны регистрации (CH, DE, GB, BE, IT, US...; EU только если страна не установлена)
5.  status           ACTIVE (юрлицо верифицировано) | PROVISIONAL (не верифицировано/неполно)
6.  verified_legal   true | false — подтверждено ли юрлицо в публичном реестре
7.  kyb_level        L0 нет | L1 домен/контакт | L2 юрлицо+рег.номер | L3 полная регуляторика (GMP/ISO 13485/EU-GMP/FDA-registration)
8.  trust_ceiling    целое 0–100: предел доверия по верификации (L0≈65, L1≈72–75, L2≈85–90, L3≈95–100)
9.  tests_count      число публично доступных независимых проверок/инспекций/сертификатов (0 если неизвестно)
10. updated_days_ago дней с последней верифицируемой даты (инспекция/сертификат/новость; целое)
11. dominant_factor  ОСТАВЬ ПУСТЫМ — движок проставит (или ориентир: обычно QEF/PCF)
12. is_balanced      ОСТАВЬ ПУСТЫМ — движок проставит
13. score            ОСТАВЬ ПУСТЫМ — Trust Score считает движок из факторов. Не выдумывай.
14. tier             ОСТАВЬ ПУСТЫМ — движок проставит.
15. contact_website  https://...
16. contact_email    публичный email или пусто
17. contact_telegram @handle или пусто (для CDMO почти всегда пусто)
18. factor_QEF       0–100 — Качество (ВЕС 35%, главный): GMP-статус, ISO 13485, EU-GMP/FDA-registration, аккредитации стерильного производства
19. factor_PCF       0–100 — Контроль процесса/QMS (ВЕС 30%): валидация асептики, lyo-циклы, media fill, задокументированные SOP/QMS
20. factor_SCIF      0–100 — Целостность цепочки (ВЕС 20%): лотовая привязка финишного флакона к исходному API-лоту; прослеживаемость; cold-chain
21. factor_TRF       0–100 — Репутация/трек-рекорд: годы на рынке, публичные клиенты, история инспекций FDA/EMA
22. factor_FRF       0–100 — Фулфилмент (для CDMO второстепенно): мощности/сроки/география; заполняй только при данных
23. factor_CCF       0–100 — Комплаенс: регуляторные допуски, аудит-готовность
24. factor_CVF       0–100 — Подтверждения клиентов (общие): публичные референсы/кейсы
25. factor_CVF_B     0–100 — CVF вариант B (ВЕС 15%): ВЕРИФИЦИРОВАННЫЕ B2B-подтверждения (бренд/вендор/фарма публично заявляет это CDMO как поставщика; рамочные договоры; пресс-релизы). Заполняй ТОЛЬКО при наличии такого подтверждения.
26. factor_RF        0–100 — Резидуальный/риск-сигнал: чем выше — тем лучше (меньше негативных индикаторов: warning letters, отзывы, recalls)
27. provisional_reason  если status=PROVISIONAL — что именно не верифицировано и какой шаг KYB нужен
28. _source_url      основной публичный URL-доказательство (реестр, EudraGMDP, FDA establishment, страница сертификатов)
29. _data_completeness  целое 0–100: насколько полно заполнена строка доказанными данными
30. _confidence      high | medium | low
31. _notes           рег.номера; адрес площадки розлива; GMP-лицензия/номер; ISO 13485 cert; типы lyo-мощностей; инциденты; источники

ШКАЛА ФАКТОРОВ (0–100) И КАЛИБРОВКА — относительно якорей файла
═══════════════════════════════════════════════════════════════════════

СВЯЗИ КОНТРАГЕНТОВ (отдельным блоком ПОСЛЕ CSV — НЕ в основной CSV)
═══════════════════════════════════════════════════════════════════════
FILL_FINISH сидит МЕЖДУ заводом-синтеза (bulk API) и витриной/брендом. Если
публично подтверждено «кто чей API розливает» или «чей продукт фасует CDMO» —
предложи рёбра (направление downstream→upstream, «покупает у»):
   buyer_domain ; supplier_domain ; lot_share(0–1) ; blind(true/false) ; источник
Примеры моделей (вноси ТОЛЬКО при публичном подтверждении):
   fillfinish_domain ; manufacturer_domain ; lot_share ; blind ; URL   (CDMO берёт bulk API у завода)
   vendor_domain ; fillfinish_domain ; lot_share ; blind ; URL          (витрина берёт готовые флаконы у CDMO)
Это даёт цель для V1 Vᵢ-пропагации (наследование верифицированного качества по цепочке).
```
