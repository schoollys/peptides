Граница нейтралитета

Платёжный провайдер обслуживает только B2B-выручку (подписки/отчёты для «решающих»). Никаких платежей за сделки участников, эскроу или оплаты за влияние на Score — это вне scope (Scope Document §06).

05 — Отказоустойчивость интеграций
СервисСтратегия при отказе
L2 RPCОчередь anchoring с ретраями; деградация — отложенный анкоринг, read не страдает; резервный RPC-провайдер
KYB/KYCОнбординг ставится в ожидание (не блокирует read); fallback-провайдер по юрисдикциям
KMS/HSMHA-конфигурация; без подписи — бейджи отдаются из кэша (TTL), новые операции ждут
CDNOrigin shielding; деградация на прямой origin при сбое edge
Email/PushОчередь с ретраями; некритично для read-пути
Платёжный провайдерИзолирован; отказ влияет только на B2B-биллинг, не на ядро

Принцип: ни одна интеграция (кроме БД/кэша) не должна ронять критичный read-путь. Запись/пересчёт/anchoring — асинхронны и переживают временные сбои внешних сервисов через очереди.
