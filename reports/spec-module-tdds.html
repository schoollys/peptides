02 — Scoring Engine
АспектДетали
ОтветственностьЧистый расчёт TS по входам; детерминирован; без I/O внутри ядра
Входfactor_inputs, weights[role], flags, algoVersion, now
Выход{score, contributions[], dominant, tier}; ScoreEvent для персистенции
Точки отказаОтсутствующие данные (→0), нарушение ΣW=1 (валидация), деление на ноль (guard)
ТестыGolden-датасеты; границы Fᵢ; balanced; no-TS; provisional
 
module scoring:
  pure compute(inputs, weights, flags, algoVersion, now) -> ScoreResult
    assert abs(sum(weights)-1) < eps         # инвариант
    contributions = factor -> Fi*Wi*Di*Vi    # см. Algorithm Spec §10
    raw = sum(contributions)
    (Ps, tier) = penalty(flags, tierOf(raw))
    return { raw*(1-Ps), contributions, dominant(contributions), tier }
  # I/O (загрузка входов, запись события) — в адаптере, не в ядре
03 — Propagation Graph
АспектДетали
СтруктураОриентированный граф; рёбра = counterparty_links (from→to, lot_share, blind, λ)
АлгоритмOne-hop взвешенная агрегация субскоров; исключение циклов; штраф концентрации; cap
Циклыvisited-множество по входящим рёбрам фактора; ребро, замыкающее цикл, исключается
СложностьO(Σ deg) на проход; граф разрежен
Точки отказаОчень высокая степень узла (ограничение/постранично), осиротевшие связи
ТестыЦикл A→B→A (запрет), концентрация top>0.6 (штраф), blind (λ), мульти-источник (max)
 
function recomputePropagation(batchNodes):
  for node in batchNodes:                    # суточный batch
    for factor in FACTORS:
      edges = incoming(node, factor)
      edges = dropCycleClosing(edges, node)  # DFS/visited
      Sprop = weightedAvg(edges, subscoreDamped30d)  # демпфер
      vi = applyConcentrationAndCap(blend(base(node), Sprop), edges)
      persist(node, factor, vi)
  scheduleScoreRecompute(batchNodes)
04 — Collusion-ring Detector
АспектДетали
ЦельВыявление колец взаимной накрутки Vᵢ/отзывов
СигналыПлотные взаимные ссылки, общий real_identity_id, всплеск регистраций/отзывов, временные паттерны
МетодПоиск сильно связанных компонент / плотных подграфов; скоринг подозрительности
ДействиеЗаморозка пропагации по кластеру; флаг на ревью; эскалация (RB-12)
ТестыСинтетическое кольцо (детект), легитимный хаб (без ложного срабатывания)
 
function detectRings(graph):
  components = stronglyConnected(graph)      # подозрение на циклы
  for c in components where size(c) >= 2:
    score = densityScore(c) + identityOverlap(c) + burstSignal(c)
    if score > THRESHOLD:
      freezePropagation(c); raiseReview(c)
05 — Anchoring Worker
АспектДетали
ЦельБатч-якорение хешей событий в L2 через HashAnchor
ПотокСбор pending → Merkle-root → tx → ожидание подтверждений → anchored
ИдемпотентностьbatchId; повторная отправка не дублирует
ОтказыRPC down → failed → ретраи backoff (SM anchoring); read не блокируется
ТестыУспех, таймаут+ретрай, идемпотентный повтор, переключение сети
 
function anchorBatch():
  pending = selectPending(limit)
  if empty: return
  root = merkleRoot(pending.hashes)
  tx = hashAnchor.anchor(root, batchId)      # абстракция сети
  on confirmed(N blocks): markAnchored(pending, tx)
  on error/timeout: markFailed(pending); scheduleRetry(backoff)
06 — Oracle / COA Verifier
АспектДетали
ЦельПроверка подписи COA и приёмка вклада
ШагиПроверить цепочку сертификата → не в CRL → дата подписи vs компрометации → анти-reuse media
ОтказыНеаккредитованный (reject), отозванный (CRL_REVOKED), reuse (ARTIFACT_REUSE)
ТестыSEC-S2, SEC-A4, SEC-A5 (см. Security Test Cases)
 
function verifyCOA(coa):
  if not validChain(coa.cert): reject(BAD_SIGNATURE)
  if revoked(coa.cert, atDate=coa.signedAt): reject(CRL_REVOKED)
  if mediaSeen(coa.mediaHash): reject(ARTIFACT_REUSE)
  accept(coa) -> factor_input(Vi by custody independence)
