03 — Зоны доверия
// Trust zones (снаружи → внутрь) [Публичный интернет] │ TLS, WAF, anti-DDoS ▼ [DMZ: CDN / Edge / API Gateway] — только публичный read + аутентиф. write │ mTLS внутри периметра ▼ [App-зона: монолит + worker] — бизнес-логика, нет прямого доступа извне │ ├─► [Data-зона: PostgreSQL/Redis/Storage] — приватная сеть, без публичного IP └─► [Crypto-зона: signing-сервис → KMS/HSM] — строжайший доступ, аудит

Сегментация сети: каждая зона доступна только из соседней внутренней; data- и crypto-зоны не имеют публичных адресов.

05 — PKI-иерархия
Root CA (offline, в HSM) ├─► Intermediate CA: Oracles ──► сертификаты лабораторий-оракулов ├─► Intermediate CA: Registry ──► ключ подписи бейджей, countersign lot-keys └─► CRL / статус-сервис ──► отзыв скомпрометированных/деаккредитованных
  • Root CA — оффлайн, в HSM; используется только для подписи промежуточных.
  • Оракулы получают сертификаты через аккредитацию; ротация по сроку.
  • Отзыв (CRL): компрометация/деаккредитация → COA оракула становятся недействительны, затронутые Score пересчитываются (BR-COA-PKI).
  • Lot-QR countersign реестром (Волна 2) — привязка артефактов к партии.

Связь с приватностью

ПДн минимизируются (храним статус, не сырьё KYB — ADR-007). В ончейн и логи ПДн не попадают. Удаление — tombstone: ПДн снимаются, публично значимая история сохраняется (BR-OFFBOARD, BR-RETENTION).

07 — Шифрование
  • В транзите: TLS снаружи, mTLS между внутренними сервисами.
  • В покое: шифрование БД, Object Storage, бэкапов; ключи шифрования — в KMS.
  • Подписи: бейдж, COA-верификация, anchoring — через signing-сервис/HSM (ADR-008).
  • Ончейн: только hash(payload‖salt), соль не публична — анонимизация (ADR-005).
08 — Threat model

Полная threat model (STRIDE + 8 нишевых abuse-сценариев с контрмерами: накрутка отзывов, sample-swap, collusion-ring, Sybil, подделка бейджа, манипуляция пропагацией, атака на апелляции, компрометация оракула) — в Волне 4. Контрмеры на уровне правил — в Business Rules (BR-VAL-INDEP, BR-PROP-*, BR-PS-SWAP, BR-COA-PKI и др.).

Сводка безопасности

Безопасность стоит на: сегментации зон, изоляции ключей (KMS/HSM), PKI с отзывом, минимизации ПДн, анонимных ончейн-хешах и антиманипуляционных правилах скоринга. Юридический слой (ToS/право) — отдельно в P0; здесь — техническая часть.
