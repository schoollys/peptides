00 — Контекст и принятые решения

P1: финализация ядра алгоритма

Три последних формальных пробела ядра из аудита полноты. Структура TS = Σ(Fᵢ·Wᵢ·Dᵢ·Vᵢ)·(1−Pₛ) не меняется — уточняются Dᵢ (распад) и Vᵢ (доверенность) для случаев, ранее заданных качественно.

01 — Dᵢ по классам распада

Разные данные стареют с разной скоростью

Единая кривая искажала бы и долгоживущие факты (сертификация), и короткоживущие (логистика конкретной поставки). Вводим 4 класса с периодом полураспада; внутри класса распад непрерывный.

Формула распада

Dᵢ = clamp(floor_class, 1.0, 0.5 ^ (age_days / t_half_class)) // непрерывный экспоненциальный распад; 0–30 дней ≈ почти 1.0 для всех классов

Классы provisional v0.9

КлассТипы данныхt½ (дн)FloorОбоснование

B — КачествоCOA/лаб-тесты чистоты, раскрытие провенанса (SCIF)1200.20Качество дрейфует от партии к партии — нужно обновлять

Глобальная банд-таблица v2 (0–30→1.0 … >365→0.15) остаётся дефолтом для сигналов без явного класса. t½ и floor — под валидацию SME (чек-лист §03 закрытия P0).

Примеры

COA возрастом 120 дн (класс B): Dᵢ = 0.5^(120/120) = 0.50.

GMP-сертификат 365 дн (класс A): 0.5^(365/365) = 0.50 (выше floor 0.30) → 0.50.

IoT-лог поставки 45 дн (класс C): 0.5^(45/45) = 0.50, но через 180 дн → 0.5^4 = 0.0625 → floor 0.10.

Почему классы, а не per-signal

4 класса покрывают реальную разницу горизонтов (сертификат vs поставка) без раздувания калибровки до десятков отдельных half-life. Каждый сигнал маппится в класс; при необходимости класс переопределяется на конкретный фактор. Баланс точности и поддерживаемости.

02 — Агрегация пропагации Vᵢ

Перетекание доверия от нескольких контрагентов

Когда участник раскрывает несколько источников по фактору, их субскоры сворачиваются в надбавку к Vᵢ — но так, чтобы нельзя было «спрятаться» за одним сильным источником и чтобы непокрытая доля не получала бонус.

Обозначения

  • kₙ — раскрытые и верифицированные контрагенты по фактору; sₖ — релевантный субскор контрагента (0–100); wₖ — доля партий участника от k.
  • coverage = Σwₖ — доля партий с раскрытым+верифицированным источником (остаток — на базовом Vᵢ=0.40).
  • top_share — доля крупнейшего источника среди раскрытых (концентрация).

Формула

S_prop = Σ(wₖ·sₖ) / Σ(wₖ) // средний субскор, взвеш. по доле партий δ = CONC_PENALTY (0.90) если top_share > 0.60, иначе 1.0 // плоский штраф за концентрацию (как в spec-algorithm) V_prop = [0.40 + (S_prop/100)·0.55] · λ_blind · δ // λ_blind=0.85 для blind, иначе 1.0 V_prop = min(V_prop, 0.85) // cap: пропагация слабее прямого blind-теста (1.0) Vᵢ(фактор) = coverage·V_prop + (1 − coverage)·0.40 // непокрытые партии = базовый 0.40

Порог концентрации (0.60), CONC_PENALTY (0.90) и cap (0.85) — provisional v0.9. Циклические/взаимные связи в пропагацию не входят (раздел 05 v2); субскоры берутся как демпфированное 30-дн среднее (v2).

Пример

Вендор: источник A (s=80, доля 0.5), источник B (s=70, доля 0.3), нераскрыто 0.2. Не blind.

coverage = 0.8; S_prop = (0.5·80 + 0.3·70)/0.8 = 61/0.8 = 76.25.

top_share = 0.5/0.8 = 0.625 > 0.60 → δ = CONC_PENALTY = 0.90.

V_prop = (0.40 + 0.7625·0.55)·1.0·0.90 = 0.819·0.90 = 0.737 (≤0.85 ок).

Vᵢ = 0.8·0.737 + 0.2·0.40 = 0.590 + 0.08 = 0.670.

Что обеспечивает формула

Покрытие не даёт «один Gold-источник прикрыть скрытый объём»; штраф за концентрацию вознаграждает диверсификацию (несколько проверенных источников надёжнее одного); cap гарантирует, что косвенное доверие через контрагента всегда слабее прямого независимого теста.

03 — Комбинирование Vᵢ из нескольких источников

Один факт подтверждён источниками разной доверенности

Когда один и тот же фактор подтверждён несколькими независимыми источниками (blind-тест, verified-buyer, self), итоговую доверенность определяет сильнейшее независимое подтверждение — а не их смесь.

Правило

Vᵢ = max(V_source) по ВЗАИМНО НЕЗАВИСИМЫМ источникам // независимость обязательна: COA вендора + самодекларация вендора = НЕ два источника Если источники ПРОТИВОРЕЧАТ (напр. blind-тест fail vs vendor-COA pass): → значение фактора замораживается до разбора → при признаках подмены — инцидент Pₛ (sample-swap, раздел 02 v2) → НЕ берём молча max

Почему max, а не среднее

ПодходПроблема
Среднее/смесьСлабый источник (self 0.40) «разбавлял» бы сильное независимое подтверждение (blind 1.0) — наказание за наличие доп.данных
МинимумНаличие любого слабого источника обрушивало бы доверие — демотивирует подавать что-либо сверх минимума
max (принято)Сильнейшее независимое подтверждение определяет доверие; слабые источники не вредят, но и не поднимают выше своего independent-потолка
Пример

QEF подтверждён blind-тестом (V=1.0) и фото verified-buyer (V=0.75), результаты согласованы → Vᵢ = max(1.0, 0.75) = 1.0.

Если blind-тест противоречит vendor-COA → фактор замораживается + расследование на sample-swap, а не усреднение.

Разведение с другими правилами

Это не то же, что мульти-оракульный cap Fᵢ (Волна 1 §03): тот требует ≥2 источника, чтобы высоковесовой фактор вообще засчитался. Здесь — какой Vᵢ применить, когда источники уже есть. И не то же, что пропагация (§02): там Vᵢ растёт от субскора контрагента, здесь — от прямых подтверждений самого факта.

04 — Что дальше

Ядро алгоритма полностью формализовано

P1-формализация закрывает последние качественные места ядра:

  1. Dᵢ (§01) — 4 класса распада с half-life + непрерывная формула; глобальная таблица как дефолт.
  2. Пропагация Vᵢ (§02) — взвешивание по долям + насыщение покрытием + штраф за концентрацию + cap.
  3. Комбинирование Vᵢ (§03) — max по независимым источникам + заморозка при противоречии.
Статус ядра — готово к сборке PRD

Все компоненты TS определены формулами: Fᵢ (9 факторов, Волна 1 §04–07), Wᵢ (матрица 22×9, §08), Dᵢ (классы распада, здесь §01), Vᵢ (оракулы + пропагация §02 + комбинирование §03), Pₛ (каталог + свёртка + потолок тира, v2). Остаётся подстановка чисел (валидация SME, чек-лист P0 §03) — структура завершена.

Статус на дату ревизии: описанный ниже «следующий шаг» (сборка единого PRD, закрытие P1 право/идентичность/экономика и P2-приложений) впоследствии выполнен — это исторический phase-closure отчёт. Актуальный сводный статус — в PRD.

Следующий шаг — сборка единого PRD-документа из всех волн + закрытие оставшихся P1 (право/идентичность/экономика) и P2-приложений. Все числовые коэффициенты — provisional под валидацию SME/комитета.
