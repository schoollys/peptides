Распределение бэклога по спринтам и фазам с привязкой к KPI-выходам. Опирается на Roadmap (Vision) и зависимости из WBS.

Provisional

План построен на референсной команде и относительных оценках. Это сценарий для планирования, не обязательство. Реальные даты появятся после грумминга и 1–2 спринтов калибровки velocity.

База оценок SP

SP спринтов даны на уровне задач и включают сквозные дорожки (инфра/DevOps, безопасность, QA, hardening, e2e, стабилизация) — поэтому суммы по фазам (MVP ~90, V1 ~87) ожидаемо превышают story-роллап из Backlog §04 (MVP ~47, V1 ~44), который считает только уровень историй. Это не противоречие, а разные уровни декомпозиции; velocity (~18–22 SP/спринт) откалибрована именно к task-уровню этого плана.

04 — Майлстоуны и release gate
M1 — MVP (≈после S5): публичный каталог, Score+бейдж, онбординг, оракулы/ончейн. Exit-gate: KPI Bootstrap/MVP (Волна 3), traceability 100% для AC MVP, 0 S1/S2.
M2 — V1 (≈после S10): пропагация, апелляции, custody, B2B, Chain Pull. Exit-gate: KPI Growth, security-audit пройден, нагрузочное в пределах NFR.
M3 — V2 (S11+): ZK-приватность, аналитика/VaaS, охват. Exit-gate: приватность подтверждена, аналитика без реидентификации.

Каждый release gate = Exit Criteria из Test Plan §03 + DoD из DoR/DoD.
