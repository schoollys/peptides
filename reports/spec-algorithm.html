Algorithm Specification

Единый авторитетный документ по расчёту Trust Score. Консолидирует математику, разбросанную по Wave 1, P1-алгоритм и Business Rules, в форму, по которой движок реализуется без додумывания. Все числа — provisional v0.9 до SME (источник значений: Parameters Registry).

02 — Мастер-формула
TS = ( Σᵢ Fᵢ · Wᵢ · Dᵢ · Vᵢ ) · ( 1 − Pₛ )
ТермДиапазонСмысл
Fᵢ0–100Внутреннее качество по фактору i (абсолютные полосы)
Wᵢ0–1, ΣWᵢ=1Вес фактора для роли (матрица 22×9, по algo_version)
Dᵢfloor–1Свежесть (экспоненциальный спад по классу)
Vᵢ0–1Верифицируемость источника (с пропагацией)
Pₛ0–<1Штрафной множитель (с потолками по тиру)

Итоговый TS ∈ [0,100]. Отображается с доминирующим фактором (§09) и тиром (platinum/gold/silver/bronze/watch; пороги — см. Parameters Registry) либо статусом new/provisional при недостатке данных.

03 — Fᵢ: нормализация (9 факторов)

Принципы: абсолютные полосы (не относительный ранг); отсутствующие данные = 0 (не штраф, а отсутствие вклада); мульти-оракульный потолок. Полные таблицы полос — в Wave 1.

function normalizeFactor(factor, rawInputs, algoVersion):
  bands = PARAMS[algoVersion].bands[factor]   # абсолютные полосы
  if rawInputs is empty: return 0              # missing = 0
  raw = aggregateRaw(factor, rawInputs)        # напр. медиана чистоты
  Fi  = mapToBand(raw, bands)                  # 0..100
  # мульти-оракульный потолок: без ≥N независимых источников — cap
  if independentSources(rawInputs) < bands.minSources:
      Fi = min(Fi, bands.cappedMax)
  return clamp(Fi, 0, 100)
04 — Dᵢ: затухание свежести
Dᵢ = max( floor, 2^( −Δt / T½(class) ) )
КлассT½Применяется к
A365 днМедленно меняющееся (процесс, лицензии)
B120 днКачество синтеза, supply-chain
C45 днЛогистика, cold-chain, отзывчивость
D90 днПодтверждения покупателей
 
function decay(observedAt, class, now):
  dtDays = (now - observedAt).days
  d = pow(2, -dtDays / HALF_LIFE[class])
  return max(FLOOR[class], d)

Δt — возраст наблюдения. floor не даёт вкладу занулиться полностью (история частично значима).

05 — Vᵢ: множитель верификации
Vᵢ(base) = 0.40 + ( Sub_counterparty,factor / 100 ) · 0.55

Базовый Vᵢ зависит от субскора контрагента-источника по тому же фактору. Для blind-связей множится на λ_blind = 0.85. Самостоятельный сэмплинг вендора никогда не даёт Vᵢ=1.0.

function baseVi(sourceSubscore, isBlind):
  vi = 0.40 + (sourceSubscore/100) * 0.55
  if isBlind: vi *= LAMBDA_BLIND        # 0.85
  return clamp(vi, 0, 1)
06 — Пропагация Vᵢ (one-hop, взвешенная)

Bounded одношаговая пропагация субскора контрагентов в Vᵢ. Запрещены циклы; штраф за концентрацию источников; жёсткий потолок.

S_prop = Σⱼ ( lot_shareⱼ · subⱼ ) / Σⱼ lot_shareⱼ   →   Vᵢ = min( 0.85, blend(base, S_prop) ) · concPenalty
function propagatedVi(node, factor, graph):
  neighbors = graph.incoming(node, factor)
  neighbors = excludeCycles(neighbors, node)        # BR-PROP-CYCLE
  if neighbors is empty: return baseViOnly(node)
  num = 0; den = 0; shares = []
  for e in neighbors:
      s = e.lotShare * lambdaFor(e)                 # λ_blind для blind
      num += s * subscore(e.from, factor)
      den += s; shares.append(s)
  Sprop = num / den
  vi = blend(baseViOnly(node), Sprop)               # по покрытию
  topShare = max(shares) / den
  if topShare > CONC_THRESHOLD:                      # 0.60
      vi *= CONC_PENALTY                             # штраф концентрации
  return min(VI_CAP, vi)                             # 0.85
Мульти-источник

Если фактор подтверждён несколькими независимыми источниками — берётся max Vᵢ (сильнейшее независимое подтверждение). При конфликте источников — заморозка/расследование, а не усреднение.

07 — Pₛ: штрафной множитель
Pₛ = 1 − ∏ₖ ( 1 − pₖ )   (с жёсткими потолками тира)
function penalty(flags, tier):
  prod = 1
  for f in flags where f.status == 'upheld' and not expired(f):
      prod *= (1 - f.pk)
  Ps = 1 - prod
  # жёсткий потолок: при CRITICAL-флаге тир не выше cap
  if anyCritical(flags): tier = min(tier, TIER_CAP_ON_CRITICAL)
  return Ps, tier

Подтверждённые флаги хранятся на срок наказания, затем tombstone. INFO-флаги информативны и не входят в произведение до подтверждения.

08 — Демпфер (30-дн скользящее среднее + отмена буфера)

Субскор контрагента в пропагации берётся как 30-дневное среднее — резкое падение у поставщика не обрушивает Score мгновенно. НО: если после алерта участник продолжает сорсинг у флагнутого контрагента — буфер аннулируется, пересчёт немедленный.

function dampedSub(counterparty, factor, now):
  return movingAverage(subscoreSeries(counterparty, factor), window=30d)

function onAlert(node, flaggedCounterparty, severity):
  notify(node, severity)                             # INFO/WARNING/CRITICAL
  if continuesSourcing(node, flaggedCounterparty, afterAlert=true):
      cancelBuffer(node, flaggedCounterparty)        # отмена 30-дн сглаживания
      recomputeNow(node)                             # немедленный пересчёт вниз
09 — Доминирующий фактор
function dominantFactor(contributions):
  # contribution_i = Fi*Wi*Di*Vi
  total = sum(contributions.values)
  if total == 0: return null                         # no-TS/provisional
  leader = argmax(contributions)
  share  = contributions[leader] / total
  if share < BALANCED_THRESHOLD: return BALANCED     # сбалансированный профиль
  return leader

Отображение: «82 — преимущественно качество» (по leader) либо «82 — сбалансированный профиль». Глоссарий 9 факторов — в v2-документе.

10 — Полный псевдокод пересчёта Score
function computeTrustScore(participant, now, algoVersion):
  P = PARAMS[algoVersion]
  weights = P.weights[participant.role]              # ΣW = 1
  contributions = {}
  for factor in FACTORS:
      inputs = loadInputs(participant, factor)
      Fi = normalizeFactor(factor, inputs, algoVersion)
      Di = decay(latestObservedAt(inputs), classOf(factor), now)
      Vi = propagatedVi(participant, factor, graph)  # включает damper, λ, cap
      contributions[factor] = Fi * weights[factor] * Di * Vi
  rawScore = sum(contributions.values)
  flags = loadUpheldFlags(participant)
  Ps, tier = penalty(flags, tierFromScore(rawScore))
  score = rawScore * (1 - Ps)
  dom = dominantFactor(contributions)
  emit ScoreEvent(participant, score, dom, algoVersion,
                  inputsHash=hash(inputs), anchor=schedule)
  return {score, dom, tier, algoVersion}

Триггеры: критические события (флаг, новый независимый тест, отмена буфера) → пересчёт немедленно; decay/пропагация → суточный batch (см. TDD).
