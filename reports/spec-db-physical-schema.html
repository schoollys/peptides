Физическая схема БД

PostgreSQL 15+. Реализация логической ERD (ERD & Security) в физические таблицы с типами, ограничениями и индексами. Рабочий DDL — отдельным файлом.

📄 db/schema.sql — рабочий DDL
Принципы схемы

Off-chain хранение (в L2 — только хеши, ADR-005); ПДн минимизированы (KYB-сырьё у провайдера, ADR-007); algo_version — forward-only; tombstone сохраняет публичную историю при очистке ПДн.

05 — Инварианты и ограничения
  • SUM(weight) по (algo_version, role_code) = 1.000 — проверяется приложением/тестом (BR-SCORE-2).
  • value (Fᵢ) ∈ [0,100]; vi, pk, lambda ∈ [0,1] — CHECK-ограничения.
  • counterparty_links: from_id ≠ to_id, уникальность пары; циклы исключаются логикой пропагации (BR-PROP-CYCLE).
  • Анти-reuse: уникальный media_hash на COA.
  • score_events.algo_version неизменна после записи — forward-only (BR-VERSION).
06 — Стратегия миграций
  • Forward-only, обратносовместимые: добавление колонок/таблиц до удаления старых (expand → contract), чтобы откат кода не ломал данные.
  • Версионируемые миграции в репозитории; применяются автоматически на staging, с approve на prod (Infra & DevOps §04).
  • Изменение параметров скоринга → новая запись в algo_versions + новые role_factor_weights; исторические score_events остаются под своей версией.
  • Tombstone (оффбординг): status='tombstone' + очистка ПДн; публичная история (score_events, penalty_flags) сохраняется (BR-OFFBOARD/RETENTION).
  • Партиционирование score_events/factor_inputs по времени при росте; шардинг по диапазону id (ADR-002).
