Tech Stack Map

Единый справочник техстека — одна точка истины по технологиям всех слоёв. Каждый выбор обоснован в ADR. Все позиции зафиксированы (Accepted); конкретные версии — LTS на инициализации репозитория.

02 — Слои стека
Клиент / SSR ── Next.js (React, SSR/RSC) · TypeScript
    │
API / монолит ── NestJS @ Fastify · 13 доменных модулей · TypeScript
    │    ├── Scoring ядро (чистые функции, фреймворк-независимо)
    │    └── Signing-сервис (бейдж/лоты) → KMS/HSM
Воркеры ── BullMQ @ Redis (пересчёт · пропагация · anchoring · outbox)
    │
Данные ── PostgreSQL 15 (primary) · Redis (cache/queue) · Object Storage (S3/MinIO)
    │
Цепь / внешние ── L2 Base/Arbitrum (HashAnchor) · KYB/KYC-провайдер · SMS/OTP
    │
Платформа ── Docker · managed-контейнеры (k8s-ready) · Terraform · EU/CH

Открыто (P1): выбор вендоров

Зафиксированы архитектурные решения, не конкретные вендоры. Открытыми P1-вопросами остаются: облачный провайдер (регион EU/CH задан — ADR-003), CI-платформа, observability-вендор, конкретные провайдеры KYB/KYC и KMS/HSM. Требования к ним инвариантны к выбору вендора — см. Integration Map и Infra & DevOps.

05 — Принципы выбора
ПринципСледствие
Единый языкTS фронт+бэк+воркеры → меньше контекста для lean-команды
Минимум инфраструктуры на стартеRedis служит и кэшем, и очередью; managed-контейнеры до k8s
Границы под эволюциюМодули NestJS → безболезненный вынос в сервисы
Ядро без фреймворкаСкоринг — чистые детерминированные функции (тестируемость, регрессия)
ВоспроизводимостьIaC + контейнеры → паритет dev/stage/prod, аудит
Без лок-ина в критичномL2/KYB/KMS — за абстракциями (Integration Contracts)
