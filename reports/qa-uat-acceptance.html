UAT & Acceptance Scripts

Сценарии пользовательской приёмки по фазам — пошаговые скрипты для проверки готового продукта «глазами пользователя». Дополняют автоматический Test Plan (там TC↔AC) ручной приёмкой ключевых путей и служат release-gate.

02 — Процесс UAT
ЭтапСодержание
ВходRC на staging; все TC фазы зелёные; seed-данные загружены
УчастникиPO + представители ролей (тест-персоны из Vision) + QA
ПрогонСкрипты ниже; фиксация фактического результата vs ожидаемого
ДефектыS1/S2 блокируют sign-off; S3/S4 — в бэклог
ВыходSign-off PO + 0 открытых S1/S2 → release-gate пройден
03 — Приёмка MVP

UAT-M1 — Покупатель находит и проверяет вендора US-1.1/1.2/2.1

Дано: каталог с seed-участниками

Шаги: открыть каталог → фильтр по роли «RUO vendor» и тиру Gold → открыть профиль → проверить Score с доминантой → нажать «проверить ончейн»

Ожидаемо: отфильтрованный список; профиль показывает факторы/вклад/флаги; ончейн-проверка подтверждает целостность

UAT-M2 — Вендор заявляет профиль US-4.1/4.3

Шаги: claim → выбрать L2 → отправить на KYB (мок approved) → дождаться provisional

Ожидаемо: статус pending → provisional; для fallback-юрисдикции виден trust ceiling

UAT-M3 — Оракул подаёт COA, Score обновляется US-5.1/5.2

Шаги: подать подписанный COA по лоту → дождаться пересчёта → проверить anchor

Ожидаемо: вклад учтён; событие заанкорено; /verify подтверждает; неаккредитованный оракул отклоняется

UAT-M4 — Вендор встраивает бейдж US-3.1

Шаги: скопировать код бейджа → вставить на тест-страницу → кликнуть

Ожидаемо: бейдж рендерится, актуальный Score+доминанта; клик ведёт на профиль; подмена значения невалидна

04 — Приёмка V1

UAT-V1 — Пропагация и алерт с отменой буфера US-6.1/6.2

Шаги: уронить субскор контрагента → проверить алерт у вендора → продолжить «сорсинг» у флагнутого → проверить пересчёт

Ожидаемо: алерт нужного уровня; при продолжении — буфер аннулируется, Score пересчитан вниз немедленно

UAT-V2 — Апелляция через Steward-панель US-8.1

Шаги: подать апелляцию на флаг → проследить статусы → получить решение

Ожидаемо: панель сформирована (VRF+conflict-screen+chair); решение binding; loser-pays отражён

UAT-V3 — B2B мониторинг US-9.1/9.2

Шаги: оформить подписку на контрагента → инициировать изменение Score → проверить webhook-sink

Ожидаемо: доставлено score.updated с подписью; ретрай при сбое получателя

UAT-V4 — Lot-QR и custody US-10.1

Шаги: создать лот, привязать custody (неполную) → подать тест

Ожидаемо: Vᵢ<1.0 при неполной цепочке; self-sampling никогда 1.0

UAT-V5 — Chain Pull: втягивание контрагента US-7.1

Шаги: пригласить контрагента в реестр → контрагент регистрируется и проходит верификацию → проверить профиль инициатора

Ожидаемо: начислен Chain Pull-кредит инициатору, связь стала верифицируемой; у участника без зарегистрированных контрагентов отсутствие связей видно как сниженное покрытие Vᵢ
