API Reference

Человекочитаемый справочник по эндпоинтам. Источник истины — рабочий openapi.yaml (OpenAPI 3.1). Read — публично без аутентификации; write/protected — X-Api-Key + X-Signature (PKI).

📄 api/openapi.yaml — OpenAPI 3.1 (источник истины)
02 — Каталог (read, публично)
catalog
get/participantspublicMVP
Сегментированный каталог с фильтрами и курсорной пагинацией.
Query: role, tier, factor, verified_legal, q, cursor, limit(≤100) · 200: {data[], next_cursor}
get/participants/{id}publicMVP
Профиль: Score, доминирующий фактор, факторы (value/Vᵢ/Dᵢ/вклад), флаги, статус, algo_version.
200: ParticipantProfile · 404: not found
get/participants/{id}/historypublicMVP
История Score по algo_version (курсор).
03 — Бейдж и целостность
get/badge/{id}.svgpublicMVP
Подписанный SVG (theme=light|dark). Кэшируемо (Cache-Control, ETag); защита от подделки подписью эндпоинта (ADR-006).
get/verify/{anchor_hash}publicMVP
Проверка ончейн-якоря: {anchored, l2_tx, algo_version, computed_at}.
04 — Write (protected: X-Api-Key + X-Signature)
post/claimsprotectedMVP
Claim-profile / онбординг. Требует Idempotency-Key.
202: ClaimStatus(pending_verification|provisional|rejected) · 409: claim уже существует
post/submissionsprotectedMVP
Подача данных: review|test|coa|artifact; lot_qr_key, factor, value, signed_payload (COA). Пересчёт асинхронно.
202: {submission_id, status:queued} · 422: нарушение бизнес-правил (BR-*)
post/appealsprotectedV1
Подача апелляции (subject_id, flag_id, statement) → формирование Steward-панели.
05 — B2B (V1)
post/b2b/webhooksprotectedV1
Подписка на события контрагента: score.updated, flag.opened, lot.recalled. Контракты доставки — в Integration Contracts.
06 — Коды ответов
КодЗначение
200/201/202OK / создано / принято в асинхронную обработку
400Некорректный запрос (валидация формата)
401Нет/неверная аутентификация или подпись
404Не найдено
409Конфликт (напр. дубль claim)
422Нарушение бизнес-правил (см. каталог ошибок)
429Rate limit (Retry-After)

Единый формат ошибки {code, message, details, request_id} и полный каталог кодов — в API Conventions & Errors.
