Test Strategy

План тестирования, привязанный к acceptance criteria (User Stories), бизнес-правилам (Business Rules / Edge Cases) и NFR (PRD Foundations).

02 — Пирамида тестирования
/\ e2e / приёмочные (мало, по US-*) /--\ integration (API+БД+очередь) /------\ contract (OpenAPI / вебхуки) /--------\ unit (ядро скоринга — максимум)

Самый плотный слой — unit на скоринг-ядре (детерминированная математика). Сверху — контрактные и интеграционные, затем тонкий слой e2e по ключевым пользовательским путям.

06 — Security-тесты
  • SAST/DAST/dependency-scan в CI (Infra & DevOps).
  • Подписи и PKI: отказ при невалидной подписи, отозванном сертификате, replay вебхука.
  • Abuse-сценарии из threat model (STRIDE + 8 нишевых): sybil/PoP, sample-swap, collusion-ring, артефакт-reuse, манипуляция бейджем.
  • AuthZ-матрица: запрет доступа к чужим write-ресурсам.
  • Приватность: shielded-данные не утекают в публичный API/бейдж/логи.

Definition of Done

Фича готова, когда: код + тесты на затронутые BR-* + контракт обновлён (если менялся API) + проходят все CI-гейты + обновлён changelog/algo_version при изменении параметров.
