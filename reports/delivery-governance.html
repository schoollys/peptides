DoR / DoD & RACI Matrix

Правила готовности/завершённости работы и распределение ответственности по ключевым активностям. Согласовано с lean-hybrid штатом (Волна 4) и Test Strategy.

02 — Definition of Ready (история готова к взятию)
  • История в формате «Как <роль>, я хочу…, чтобы…» с ценностью.
  • Acceptance Criteria (Given/When/Then) сформулированы и однозначны.
  • Затронутые BR-* и edge cases идентифицированы.
  • Зависимости известны и разблокированы (или явно отмечены).
  • UX-макет/состояния (empty/loading/error) доступны, если есть UI.
  • Контракт API определён (если затрагивается) — соответствует openapi.yaml.
  • Оценка (SP) проставлена командой; история влезает в один спринт (иначе сплит).
  • Влияние на algo_version / приватность / безопасность оценено.
03 — Definition of Done (история завершена)
  • Код смержен; ревью пройдено; стиль/типы без ошибок.
  • Все AC истории покрыты тест-кейсами (TC-*), тесты зелёные.
  • Затронутые BR-* имеют автотест; регрессия скоринга проходит (если ядро).
  • Контрактные тесты против openapi.yaml зелёные (если API менялся).
  • Security-кейсы (SEC-*) покрывают новые поверхности; сканы без High/Critical.
  • Traceability обновлён (AC ↔ TC ↔ BR).
  • Наблюдаемость: метрики/логи/алерты добавлены для нового поведения.
  • Документация обновлена; changelog/algo_version — при изменении параметров.
  • Развёрнуто на staging; приёмка PO; нет открытых S1/S2.
Связь с релизом

Release gate (Release Plan §04) = сумма DoD всех историй фазы + Exit Criteria из Test Plan.
