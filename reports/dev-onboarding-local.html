Developer Onboarding & Local Dev Guide

Как новый разработчик поднимает проект локально и начинает коммитить в первый день. Дополняет Infra & DevOps практической стороной. Команды иллюстративны — конкретика появится при инициализации репозитория.

02 — Предпосылки
ИнструментВерсия/назначение
Node.js + pnpmLTS; стек TypeScript/Node (ADR-009)
Docker + ComposeЛокальные PostgreSQL, Redis, MinIO (object storage)
PostgreSQL 15Через Compose; схема из db/schema.sql
ДоступыРепозиторий, секрет-менеджер (dev-scope), реестр образов
03 — Quickstart
# 1. Клонировать и установить
git clone <repo> && cd peptidetrust
pnpm install

# 2. Поднять инфраструктуру (БД/кэш/storage + моки)
cp .env.example .env.local
docker compose up -d        # postgres, redis, minio, mock-kyb, mock-l2, mock-kms

# 3. Применить схему и засеять данные
pnpm db:migrate
pnpm db:seed                # синтетические участники + golden-датасеты

# 4. Запустить приложение
pnpm dev                    # web (SSR) + API + worker

# 5. Проверить
open http://localhost:3000              # каталог
curl http://localhost:3000/v1/participants

Полный список — в .env.example; значения скоринга читаются из Parameters Registry по ALGO_VERSION.

05 — Мок-интеграции
СервисМок в devПоведение
KYB/KYCmock-kybКонфигурируемый ответ verify (approved/pending/rejected)
L2 anchoringmock-l2Возвращает фейковый txHash; /verify детерминирован
KMS/HSMmock-kms / soft-HSMПодпись dev-ключом (бейджи/лоты)
SMS/OTPmock-smsКод в лог вместо реальной отправки
B2B webhooklocal sinkПринимает доставки, показывает payload
Принцип

Локально не зависим ни от одного внешнего платного сервиса. Контракты моков совпадают с Integration Contracts, чтобы поведение совпадало с prod.

06 — Seed-данные
  1. Синтетические участники по всем 22 ролям (разные тиры, статусы unclaimed/provisional/active).
  2. Golden-датасеты для скоринга — эталонные входы→ожидаемый Score (регрессия).
  3. Примеры лотов/COA/custody (валидные и заведомо «битые» для негативных тестов).
  4. Связи контрагентов для проверки пропагации (включая заведомый цикл).

Подробно — в Data Migration & Seeding.

07 — Секреты в dev
  • Никаких prod-ключей локально. Dev использует выделенные dev-ключи/моки.
  • .env.local в .gitignore; коммитится только .env.example.
  • Крипто-ключи (подпись) — soft-HSM/dev-KMS; ротация не требуется, но формат совпадает с prod (KMS/HSM).
  • Pre-commit хук со scan на утечку секретов.
08 — Рабочий цикл
# перед PR
pnpm lint && pnpm typecheck
pnpm test                 # unit + integration
pnpm test:contract        # против openapi.yaml
pnpm test:scoring         # golden-датасеты (регрессия)

Ветвление/коммиты/структура — в Coding Standards. DoR/DoD — в Governance.
