# PeptideTrust — v0 BA Prompts (бизнес-пробелы)

Назначение: закрыть бизнес-критичные пробелы прототипа (петля обращений, монетизация B2B, фокус MVP, правовой слой). Промпты на русском, готовы к вставке в v0.app по одному.
Базовый деплой: https://peptidetrust-app-setup.vercel.app
Дизайн-токены сохранять как есть: основной `#533afd`, тени `shadow-stripe-*`, tabular-nums, тиры `tier-gold #c9a227 / tier-silver #9aa0a6 / tier-bronze #b07a45 / tier-new(provisional) #5b78a8`, error `#b3261e`.

Зависимости: эти промпты предполагают, что уже применён `v0-fix-prompts.md` (особенно FIX-01 — единый модуль `lib/participants.ts`, и FIX-03 — mock-auth).

Источники истины: `README.md` (петля «Trust Score → поток обращений → сделки вне реестра»), `reports/p1-legal-identity-economics.html` (4 B2B-SKU + граница нейтралитета), `reports/prd-scope-document.html` (скоуп MVP), `reports/PRD.html`.

Жёсткое ограничение нейтралитета (соблюдать во ВСЕХ промптах): реестр — **не маркетплейс**, не участвует в сделке и не берёт комиссию. Платит **потребитель информации** («решающий»), не **объект оценки**. Никакой B2B-продукт не даёт оцениваемому участнику купить влияние на свой Score/позицию/флаги.

## P0 — блокеры ценности и выручки

### BA-01 · Петля обращений: CTA «Связаться» + редирект-трекер
```
Замкни главную петлю ценности продукта: после оценки Trust Score у покупателя должен быть путь к действию, а у реестра — учёт обращений. Сейчас профиль /p/{id} не даёт связаться с участником.

Контекст: реестр НЕ маркетплейс и НЕ участвует в сделке. Контакт происходит через внешние каналы участника; мы лишь даём ссылку и считаем клик-обращение (не контент сделки).

Требования:
- В шапку профиля /p/{id} добавь основной CTA «Связаться с участником» (primary, токен #533afd). Рядом — вторичный «Сообщить о нарушении» (ведёт на /report?participant={id}).
- По клику открывается drawer/диалог «Каналы участника» со списком из mock-данных участника (добавь в lib/participants.ts поля: website, email, telegram — заполни для 12 участников; часть каналов может отсутствовать). Каждая строка-канал — это переход через редирект-трекер /out/{id}?ch={website|email|telegram}.
- Создай страницу-редирект /out/[id]: записывает обращение в mock-лог (localStorage, см. BA-02), показывает короткий интерстишл «Вы покидаете PeptideTrust. Реестр не участвует в сделке и не несёт ответственности за коммуникацию.» + кнопка «Перейти →» (для mock — ведёт на about:blank или сам сайт участника) и «Отмена».
- Для provisional/Watch-участников CTA «Связаться» оставь активным, но добавь предупреждающий бейдж рядом (provisional — «накопление данных», Watch — красный акцент #b3261e «есть подтверждённые флаги»).
- НЕ показывай цен, корзины, «купить», эскроу — только контакт.
Не меняй визуальные токены и стиль карточек профиля.
```

### BA-02 · «Поток обращений» в кабинете участника
```
Дай участнику измеримый ROI от высокого Score: метрику входящих обращений. Это мотивация подавать данные и поднимать Trust Score.

Требования:
- На /dashboard добавь карточку-блок «Поток обращений» рядом со Score-героем. Источник — mock-лог обращений из localStorage (события, которые пишет редирект /out/{id} из BA-01); если событий нет — засей 14–30 правдоподобных событий за последние 30 дней для текущего участника.
- Покажи: число обращений за 7 / 30 дней (tabular-nums), маленький спарклайн/бар по дням, разбивку по каналам (website/email/telegram), и дельту к предыдущему периоду (зелёный рост / нейтральная).
- Добавь строку-инсайт, связывающую Score и поток, напр.: «Участники тира Gold получают в среднем в N× больше обращений, чем Silver» (mock-сравнение по тирам из lib/participants.ts).
- Приватность: показывай только агрегаты (счётчики/каналы/даты), НЕ персональные данные покупателей и НЕ содержимое переписки.
- Пустое состояние: «Пока нет обращений — повысьте полноту профиля и Trust Score, чтобы попасть выше в каталоге» с ссылкой на /submit.
Стиль — текущие токены и карточки дашборда.
```

### BA-03 · B2B-витрина 4 SKU (заменить заглушку «Скоро»)
```
B2B — единственный заявленный источник выручки, но сейчас это заглушка «Скоро». Сделай реальную витрину из 4 SKU. Биллинг НЕ нужен (без цен и оплаты на MVP) — цель валидировать спрос.

Контекст нейтралитета: платит потребитель информации («решающий»), не объект оценки; ни один SKU не позволяет купить влияние на свой Score. Аналитика — только агрегированная.

Требования:
- Сделай страницу /b2b (вернуть в навигацию её НЕ нужно — оставить как в FIX-08: ссылка в футере + вторичный CTA; сама страница перестаёт быть «Скоро»).
- Hero: заголовок «PeptideTrust для организаций», подзаголовок про модель «платит решающий, не оцениваемый», явная плашка нейтралитета.
- 4 карточки SKU (из reports/p1-legal-identity-economics.html), каждая: название, «кому», что входит, кнопка «Запросить доступ» → /b2b/request?sku={slug}:
  1) Due-diligence отчёты — Клиники, дистрибьюторы, prescribers — глубокая проверка поставщика: история факторов, цепочка, флаги, динамика.
  2) Мониторинг-API / вебхуки — B2B-покупатели, платформы — подписка на изменения Score/флагов контрагентов в реальном времени.
  3) Рыночная аналитика — Исследователи, инвесторы, регуляторы — агрегированные тренды ниши (без продажи влияния на конкретный Score).
  4) Verification-as-a-service — Маркетплейсы, форумы, площадки — интеграция бейджа/проверки в чужие платформы (API + SDK).
- Внизу — блок «Граница нейтралитета» (callout): «Платит потребитель информации, не объект оценки. Купить влияние на свой Score нельзя.»
- Без цен, тарифов и «купить». Только описание ценности + «Запросить доступ».
Стиль — текущие токены; карточки в духе существующих.
```

### BA-04 · Квалификация лида B2B (форма «Запросить доступ»)
```
Дай B2B-витрине (BA-03) механизм квалифицированного лид-захвата, чтобы собирать спрос по сегментам.

Требования:
- Создай /b2b/request: предзаполни выбранный SKU из query (?sku=), с возможностью сменить в dropdown (4 SKU).
- Поля формы: компания/организация, рабочий email, роль (select: клиника / дистрибьютор / prescriber / маркетплейс / исследователь / инвестор / регулятор / другое), ориентировочный объём/частота использования (select), краткое описание задачи (textarea), согласие на обработку (checkbox со ссылкой на /legal/privacy из BA-06).
- Валидация: компания и email обязательны; email — формат рабочей почты (не блокировать строго, но подсветить, если gmail/публичный домен — мягкая подсказка «желательно рабочий email»).
- Сабмит (mock): пишет лид в localStorage, показывает состояние успеха «Заявка принята, ID {request_id} (моноширинный). Мы свяжемся в течение 2 рабочих дней.» + кнопка «Вернуться к B2B».
- Без оплаты/тарифов. Тот же визуальный стиль, что у /submit и /claim (карточка, степпер не нужен — одна форма).
```

## P1 — фокус MVP и правовой слой

### BA-05 · Фокус MVP: гео EU/CH + 3 якорные роли
```
Сфокусируй прототип на MVP-скоупе: одна юрисдикция (EU/CH) и 3 «якорные» роли вместо всех 22. Это снижает юр-риск и даёт сквозной рабочий сценарий.

Якорные роли MVP: RUO-вендор (продавец для research-use-only), Лаборатория-оракул (источник верификации), GB-организатор (групповой закупщик / организатор совместных закупок).

Требования:
- В lib/participants.ts приведи 12 участников к EU/CH-юрисдикциям (DE, CH, NL, FR, AT…) и распредели роли так, чтобы доминировали 3 якорные; пометь поле isAnchorRole.
- В каталоге: добавь фасет/фильтр «Роль» с якорными ролями вверху; добавь баннер-плашку «MVP: EU/CH · приоритетные роли» (ненавязчивую, токен tier-new #5b78a8) с пояснением скоупа.
- Не-якорные роли НЕ удаляй, но помечай бейджем «вне MVP-скоупа (provisional)» и при открытии профиля показывай callout «Эта роль пока в режиме провизорного покрытия — полный Trust Score появится на следующих фазах».
- В /claim и /claim/kyb: при выборе не-якорной роли — мягкое предупреждение «Полная верификация для этой роли откроется позже; сейчас доступен провизорный профиль».
- Каталог по умолчанию сортируется по Score внутри якорных ролей.
Стиль — текущие токены; не ломай существующие фильтры/карточки.
```

### BA-06 · Правовой слой: дисклеймеры, юрисдикция, политики
```
Добавь видимый правовой слой — это launch-gate для серой RUO-ниши (риски defamation за низкий Score, GDPR, статус оператора). Тексты — заглушки-болванки для ревью юриста, но структура и места размещения должны быть реальными.

Требования:
- Глобальный дисклеймер в футере на всех страницах: «PeptideTrust — нейтральный информационный реестр (research-use-only). Не маркетплейс, не участвует в сделках, не даёт медицинских/юридических рекомендаций. Оценки носят информационный характер.»
- Создай страницы (без 404), в стиле существующих контент-страниц, со ссылками из футера:
  /legal/terms (Условия использования — включить пункт про информационный характер Score и порядок оспаривания через /appeals/new),
  /legal/privacy (Политика конфиденциальности — GDPR: контролёр/процессор, минимизация, право на удаление off-chain, что on-chain только хеши без ПДн),
  /legal/disclaimer (Отказ от ответственности + RUO-формулировка),
  /legal/jurisdiction (Оператор и юрисдикция — заглушка: «Оператор зарегистрирован в EU/CH; реестр ведётся в соответствии с …»).
- На профиле /p/{id}: рядом со Score — маленькая ссылка-сноска «Как считается и что это значит» → /trust-model, и «Информационная оценка, не обвинение» микро-дисклеймер у блока флагов.
- В форме /report и /appeals/new: добавь строку про правовой статус и конфиденциальность заявителя.
- Все юр-тексты помечай комментарием в коде {/* TODO: на ревью юриста */} и видимой плашкой «Черновик — на юридическом ревью» вверху каждой /legal/* страницы.
Не меняй визуальные токены.
```

## Порядок применения
1. **BA-01** (петля обращений) — приоритет №1, разрыв в ядре ценности.
2. **BA-02** (поток обращений в кабинете) — даёт измеримость и ROI.
3. **BA-03 → BA-04** (B2B-витрина → лид-захват) — валидация выручки.
4. **BA-05** (фокус MVP) — сужение скоупа до EU/CH + 3 ролей.
5. **BA-06** (правовой слой) — launch-gate перед демонстрацией стейкхолдерам.

Каждый промпт вставлять в v0.app по одному, проверяя сборку перед следующим.
